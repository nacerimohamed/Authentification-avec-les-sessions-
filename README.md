# README - Authentification avec Sessions et API Gateway

## Description
Ce projet développé avec Express.js présente un système simple d’authentification utilisant les sessions dans une application Node.js. L’utilisateur peut se connecter, accéder à une page sécurisée (Dashboard) et se déconnecter. Le projet s’inscrit également dans l’apprentissage des architectures microservices et du rôle d’une API Gateway.

---

# Objectifs pédagogiques

## Objectifs intermédiaires
1. Comprendre le rôle et l’importance d’une API Gateway dans une architecture microservices.
2. Identifier les avantages et les fonctionnalités principales d’une API Gateway.
3. Explorer les outils populaires pour créer une API Gateway.
4. Mettre en œuvre une API Gateway avec Express.js et http-proxy-middleware.

## Objectifs opérationnels
À la fin de ce cours, le stagiaire sera capable de :
- Comprendre le fonctionnement des sessions.
- Implémenter une authentification sécurisée.
- Gérer l’accès aux routes protégées.
- Créer une API Gateway pour connecter les microservices.

# Technologies utilisées
